# Database Migration Generator

## Overview
Automatically generates timestamped database migration files with proper up/down scripts.

## Instructions

### 1. Gather Migration Details
Ask the user for:
- Migration description (e.g., "add user email column")
- Target database (postgres, mysql, sqlite)
- Schema changes needed

### 2. Generate Migration Filename
Format: \`{timestamp}_{description}.sql\`

Example:
- \`20251116_add_user_email_column.sql\`
- \`20251116_create_posts_table.sql\`

Timestamp format: \`YYYYMMDDHHmmss\`

### 3. Create Migration File
Use template from \`templates/migration.sql\`:

\`\`\`sql
-- Migration: {description}
-- Created: {timestamp}
-- Database: {database}

-- Up Migration
BEGIN;

{up_sql}

-- Add your migration SQL here
-- Example:
-- ALTER TABLE users ADD COLUMN email VARCHAR(255);

COMMIT;

-- Down Migration
BEGIN;

{down_sql}

-- Add rollback SQL here
-- Example:
-- ALTER TABLE users DROP COLUMN email;

COMMIT;
\`\`\`

### 4. Generate Up SQL
Based on migration type:

**Add Column:**
\`\`\`sql
ALTER TABLE {table_name}
ADD COLUMN {column_name} {data_type} {constraints};
\`\`\`

**Create Table:**
\`\`\`sql
CREATE TABLE {table_name} (
  id SERIAL PRIMARY KEY,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
\`\`\`

**Add Index:**
\`\`\`sql
CREATE INDEX idx_{table}_{column}
ON {table_name}({column_name});
\`\`\`

### 5. Generate Down SQL
Automatically generate rollback for each up operation:

- \`ADD COLUMN\` â†’ \`DROP COLUMN\`
- \`CREATE TABLE\` â†’ \`DROP TABLE\`
- \`CREATE INDEX\` â†’ \`DROP INDEX\`

### 6. Validate SQL
Run validation script:

\`\`\`bash
bash scripts/validate_migration.sh {migration_file}
\`\`\`

Checks:
- SQL syntax validity
- Matching BEGIN/COMMIT
- Down migration exists
- Table/column names valid

### 7. Add to Migration Log
Update \`migrations/log.md\`:

\`\`\`markdown
## {timestamp} - {description}

**File:** \`{filename}\`
**Author:** {author}
**Date:** {date}

### Changes
- [List of changes]

### Rollback
\`\`\`bash
npm run migrate:down {timestamp}
\`\`\`
\`\`\`

## Output Format

\`\`\`markdown
# Migration Created

**File:** \`migrations/20251116_add_user_email_column.sql\`
**Description:** Add email column to users table

## Up Migration
\`\`\`sql
ALTER TABLE users ADD COLUMN email VARCHAR(255) NOT NULL;
CREATE INDEX idx_users_email ON users(email);
\`\`\`

## Down Migration
\`\`\`sql
DROP INDEX idx_users_email;
ALTER TABLE users DROP COLUMN email;
\`\`\`

## Next Steps
1. Review the migration file
2. Run \\\`npm run migrate:up\\\` to apply
3. Test in development
4. Commit to version control
\`\`\`

## Best Practices
- Always include down migration
- Use transactions (BEGIN/COMMIT)
- Add indexes for foreign keys
- Include timestamps in table schemas
- Test rollback before deployment
- Document breaking changes

## Common Patterns

### Adding a Table
\`\`\`sql
-- Up
CREATE TABLE posts (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id),
  title VARCHAR(255) NOT NULL,
  content TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_posts_user_id ON posts(user_id);

-- Down
DROP INDEX idx_posts_user_id;
DROP TABLE posts;
\`\`\`

\`\`\`sql

-- Down
ALTER TABLE users

\`\`\`

ALTER TABLE posts
ADD CONSTRAINT fk_posts_user
FOREIGN KEY (user_id) REFERENCES users(id);

-- Down
ALTER TABLE posts
DROP CONSTRAINT fk_posts_user;
